
Terminfo
--------

terminfo files describe termnals on unix systems.  Most modern
Linux distributions come with a library of terminfo files to
describe all the possible terminals.  Most of these files are
for old terminals and no longer of interest.  Each X terminal
may have its own terminfo file and the console also may have
a separate terminfo file.

The active terminfo file is found by indexing into the
terminfo files at one of the following locations:

  /$HOME/.terminfo/?/name
  /etc/terminfo/?/name
  /lib/terminfo/?/name
  /usr/share/terminfo/?/name

The "?" is first character of name, and "name" is
contents of $TERM enviornment variable.

Usually, all this is handled by libraries and is
invisible to programmers.  For "C" programs the
ncurses library has lots of documentation on terminfo.

For assembler the following functions are available in
AsmLib:

  terminfo_read  -  find and read terminfo into buffer
  terminfo_decode_setup - setup key tables
  terminfo_key_decode1 - decode key using terminfo adjusted table
  terminfo_key_decode2 - decode key using terminfo adjusted table 
  terminfo_key_extrace - get code for single key
  terminfo_extract_flag - get flag values from terminfo
  terminfo_extract_number -  get number value from termifo


Why do we need terminfo?
------------------------

Special keys on the keyboard and unusual terminals present
problems for programmers. We need to know the special codes
to describe the keys and we need to know what capibilities
a terminal has.  All this information is available in the
terminfo file.

For example the F1 key often generates the codes:
  db 1bh,4fh,50h
but it may generate other codes and the keys generated
in a console may not match the ones in x terminals.
To sort all this out, each program needs to read the
current terminfo and find the codes being used.

Where can I find more information?
----------------------------------

The ncurses source has the most information on terminfo.
For assembler programmers, studying the AsmLib functions
will probably be easier.

Also, the man page for terminfo is useful.

The term.h file in ncurses and elsewhere has information
about how to access the contents of terminfo.


Articles and books on terminfo do not match this
documentation, who is right?
----------------------------------------------------

Documentation on terminfo is for people describing terminals.
It covers the input to a compiler (tic) and is in symbolic
format. This document covers the binary terminfo file produced
by tic. If you are working in assembler, this is the
documentation you want. Also, this documentation covers Linux
and differs slighty from unix documentation.


How does Linux select a terminfo file
-------------------------------------

When a program is started, its enviornment variable $TERM
is set to the name of the terminfo file.  When running
in the console it will equal "linux" and if running
under a xterm it will be "xterm".  This is the name
of a terminfo file currently active.

Can I read terminfo myself and extract the information?
-------------------------------------------------------

Yes, the terminfo file is in a binary format and generated
by the "tic" compiler.  It has a header and 6 sections.
All this is described in the next section.

What is the structure of a binary terminfo file?
------------------------------------------------

A generic terminfo file looks like this:

  dw 011ah		;id word
  dw name_size		;size of name section
  dw flag_size		;size of flag section
  dw number_size	;number of entries in numbers section
  dw index_size		;number of entries in index section
  dw string_size	;size of string section
  times name_size db ' ';name section - filled with name strings
  times flag_size db x	;flag section - boolean byte flags
  db 0			;optional pad to force next
                         section to even address
  times number_size dw x ;index's into string section.
                          values of -1 are null entries.
  times string_size db ' ' ;size of string section
  times ? db " "	;optional extension data
